 1 # AI-Powered Shopping List Builder - Backend Specification
    2
    3 ## Overview
    4 This endpoint replaces all frontend categorization and sorting logic for shopping lists. It uses Claude
       Haiku to intelligently categorize ingredients by grocery store sections and sort them for efficient sh
      opping.
    5
    6 ## Endpoint
    7
    8 **POST** `/api/recipe-queue/build-shopping-list`
    9
   10 ## Request
   11
   12 ### Headers
   13 - `Authorization: Bearer <jwt_token>` (standard auth)
   14 - `Content-Type: application/json`
   15
   16 ### Body
   17 ```typescript
   18 {
   19   goalName?: string; // Optional, defaults to "Shopping List"
   20 }
   21 ```
   22
   23 ### Example Request
   24 ```json
   25 {
   26   "goalName": "Weekend Groceries"
   27 }
   28 ```
   29
   30 ## Response
   31

   88 ## Implementation Requirements
   89
   90 ### Step 1: Get Recipe Queue
   91 Fetch all recipes in the user's queue with full recipe data including `parsedIngredients`.
   92
   93 ```sql
   94 SELECT rq.*, r.*
   95 FROM recipe_queue rq
   96 JOIN recipes r ON rq.recipe_id = r.id
   97 WHERE rq.user_id = $1
   98 ORDER BY rq.sort_order ASC
   99 ```
  100
  101 ### Step 2: Collect and Scale Ingredients
  102 For each recipe in the queue:
  103 - Get `parsedIngredients` array
  104 - Apply `servingMultiplier` to scale amounts
  105 - Create a flat list of all ingredients
  106
  107 **Scaling Logic:**
  108 ```javascript
  109 function scaleAmount(amount, multiplier) {
  110   if (!amount || multiplier === 1) return amount;
  111
  112   // Handle fractions like "1/2"
  113   let numeric;
  114   if (amount.includes('/')) {
  115     const [num, denom] = amount.split('/').map(Number);
  116     numeric = num / denom;
  117   } else {
  118     numeric = parseFloat(amount);
  119   }
  120
  121   const scaled = numeric * multiplier;
  122
  123   // Format nicely
  124   if (scaled % 1 === 0) return scaled.toString();
  125   if (scaled < 1) {
  126     // Try common fractions
  127     const fractions = [[1,4], [1,3], [1,2], [2,3], [3,4]];
  128     for (const [n, d] of fractions) {
  129       if (Math.abs(scaled - n/d) < 0.05) return `${n}/${d}`;
  130     }
  131   }
  132   return scaled.toFixed(2).replace(/\.?0+$/, '');
  133 }
  134 ```

  443 ### Step 5: Create Tasks
  444
  445 The categorized ingredients from Claude are already sorted properly. Create tasks in that exact order:
  446
  447 ```javascript
  448 const CATEGORY_SORT_ORDER = {
  449   'Produce': 0,
  450   'Dairy': 100,
  451   'Meat': 200,
  452   'Seafood': 300,
  453   'Bakery': 400,
  454   'Canned Goods': 500,
  455   'Dry Goods': 600,
  456   'Pasta & Rice': 700,
  457   'Frozen': 800,
  458   'Beverages': 900,
  459   'Condiments': 1000,
  460   'Spices': 1100,
  461   'Baking': 1200,
  462   'Other': 9000
  463 };
  464
  465 const tasksToCreate = categorizedIngredients.map((item, index) => {
  466   // Build title
  467   const amountPart = item.amount && item.unit
  468     ? `${item.amount} ${item.unit}`
  469     : item.amount || '';
  470   const title = amountPart
  471     ? `${amountPart} ${item.ingredient}`
  472     : item.ingredient;
  473
  474   // Build notes
  475   const notes = `from ${item.recipeNames.join(', ')}`;
  476
  477   // Calculate sort order: category base + index within category
  478   const categoryBase = CATEGORY_SORT_ORDER[item.category] || CATEGORY_SORT_ORDER['Other'];
  479   const sortOrder = categoryBase + index;
  480
  481   return {
  482     title,
  483     notes,
  484     goalId: goal.id,
  485     completed: false,
  486     isToday: false,
  487     sortOrder
  488   };
  489 });
  490
  491 // Bulk create tasks (use transaction if possible)
  492 await bulkCreateTasks(tasksToCreate);
  493 ```
  494
  495 **Important:** The sortOrder ensures tasks stay grouped by category even when fetched from the database
      .

  501 ## Error Handling
  502
  503 ### Claude API Errors
  504 - **Rate Limit:** Return 429 with retry-after header
  505 - **Invalid API Key:** Return 500, log error securely
  506 - **Timeout:** Retry once, then fail with 500
  507 - **Invalid JSON Response:** Log response, return 500 with helpful message
  508
  509 ### Fallback Behavior
  510 If Claude API fails after retries, you have two options:
  511
  512 **Option A: Fail Fast** (Recommended)
  513 Return 500 error to user with message: "Failed to categorize ingredients. Please try again."
  514
  515 **Option B: Simple Fallback**
  516 Create unsorted shopping list with all items in "Other" category (not recommended as it defeats the pur
      pose).
  517
  518 ## Performance Considerations
  519
  520 ### Caching
  521 - Consider caching ingredient → category mappings for common ingredients
  522 - Cache key: `ingredient_name:unit`
  523 - TTL: 7 days
  524 - Only call Claude for uncached ingredients
  525
  526 ### Batching
  527 - Claude Haiku can handle up to ~200 ingredients in a single call
  528 - If recipe queue > 50 recipes, consider warning user or batching
  529
  530 ### Cost
  531 - Claude Haiku pricing: ~$0.25 per million input tokens, ~$1.25 per million output tokens
  532 - Average shopping list: ~100-300 tokens input, ~200-500 tokens output
  533 - Cost per request: < $0.001 (less than 1/10th of a cent)
